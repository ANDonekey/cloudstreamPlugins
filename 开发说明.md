这个仓库现在默认只发布一个可用模块：

- `JableProvider`：已经按 `https://jable.tv/` 做好的首版实现

仓库里另外还保留了两个本地参考模板，不参与发布：

- `StarterProvider`：最小起步骨架
- `ExampleProvider`：官方模板示例

## 先从这里改

1. 如果你要从零做新站点，打开 `StarterProvider/src/main/kotlin/com/starter/StarterProvider.kt`
2. 如果你要继续开发 Jable，打开 `JableProvider/src/main/kotlin/com/jable/JableProvider.kt`
3. 新站点通常先替换这几个字段：
   - `mainUrl`
   - `name`
   - `lang`
   - `supportedTypes`
4. 先实现 `search()`，确认搜索结果能出来
5. 再继续补：
   - `getMainPage()`
   - `load()`
   - `loadLinks()`

官方文档强调的顺序也是先确认能不能拿到播放链接；如果 `loadLinks()` 做不通，这个站点通常就不适合继续做下去。

## 构建命令

先确认本机使用的是 JDK 11 或更高版本，推荐 JDK 17。

Windows:

```powershell
.\gradlew.bat StarterProvider:make
```

如果你想直接使用已经配置好的本地环境，也可以运行：

```powershell
.\build.ps1 JableProvider:make
```

不传参数时，`build.ps1` 默认执行：

```powershell
.\gradlew.bat JableProvider:make
```

Jable:

```powershell
.\gradlew.bat JableProvider:make
```

## 建议开发顺序

1. 先确定目标站点和域名
2. 先确认这个站点的视频链接能不能提取
3. 用浏览器确认搜索 URL 和详情页结构
4. 在 `search()` 中把标题、封面、链接先解析出来
5. 再补 `getMainPage()`
6. 再补详情页 `load()`
7. 最后把 `loadLinks()` 接上并联调

## 对照官方文档的四个核心函数

- `search()`：搜索页解析，返回 `SearchResponse`
- `getMainPage()`：首页分区和翻页
- `load()`：详情页、剧集、推荐内容
- `loadLinks()`：真正提取播放链接，最关键

## 当前仓库里的模块说明

- `JableProvider`：当前唯一参与发布的插件
- `StarterProvider`：本地参考骨架，不参与发布
- `ExampleProvider`：本地参考示例，不参与发布
