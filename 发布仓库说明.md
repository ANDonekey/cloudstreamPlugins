# 发布仓库说明

这个模板仓库会自动生成 Cloudstream 仓库索引 JSON，不需要手写整个 `plugins.json`。

## 本地生成

先确认：

- `JAVA_HOME` 指向 JDK 17
- `local.properties` 中的 `sdk.dir` 已配置

然后执行：

```powershell
.\gradlew.bat make makePluginsJson
```

生成结果：

- 单个插件包：`各模块/build/*.cs3`
- 仓库索引：`build/plugins.json`

## 当前已验证通过的模块

- `JableProvider`

本地已成功执行：

```powershell
.\gradlew.bat JableProvider:make
```

生成文件：

- `JableProvider/build/JableProvider.cs3`

## GitHub Actions 发布方式

仓库自带工作流：

- `.github/workflows/build.yml`

这个工作流会：

1. 执行 `./gradlew make makePluginsJson`
2. 收集所有 `*.cs3`
3. 收集 `build/plugins.json`
4. 推送到 `builds` 分支

## 给 Cloudstream 使用时通常需要的地址

如果后续你把这个仓库推到 GitHub，并启用 `builds` 分支产物，常见用法是：

- 仓库 JSON 地址：`https://raw.githubusercontent.com/<你的用户名>/<仓库名>/builds/plugins.json`

用户在 Cloudstream 添加这个仓库地址后，就能拉到对应插件。
